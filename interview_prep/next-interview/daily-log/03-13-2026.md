## 📚 What I Learned

- **Duplicate detection partition** — must PARTITION BY all columns that define a duplicate (`transaction_id, policy_id, amount`), not just one column
- **DENSE_RANK for Nth salary** — always `ORDER BY salary DESC`, not by `employee_id`; and filter from the CTE, not the original table
- **Sessionization — 3-step pattern:**
  - Step 1: Use `LAG()` to detect gap > 30 min → flag `is_new_session = 1`
  - Step 2: Use `SUM(is_new_session) OVER (PARTITION BY user_id ORDER BY event_time)` as running session ID
  - Step 3: `GROUP BY user_id, session_id` → get `MIN(event_time)` and `MAX(event_time)`
- **Partitioning** — physically splits data into folders by column value (e.g. `date=2024-01-01/`); enables partition pruning to skip irrelevant folders
- **Bucketing** — hashes rows into a fixed number of files by key; helps with joins and GROUP BY on high-cardinality columns
- **Lambda Architecture** — two parallel pipelines: batch layer (accurate, slow) + speed layer (approximate, real-time); both feed a serving layer
- **Kappa Architecture** — single streaming pipeline only; replay Kafka logs to recompute history; simpler than Lambda

## 🔑 Key Patterns to Memorise

### Sessionization — Full Pattern
```sql
WITH events AS (
    SELECT user_id, event_time,
        CASE
            WHEN event_time - LAG(event_time) OVER (
                     PARTITION BY user_id ORDER BY event_time
                 ) > INTERVAL '30 minutes'
              OR LAG(event_time) OVER (
                     PARTITION BY user_id ORDER BY event_time
                 ) IS NULL
            THEN 1 ELSE 0
        END AS new_session
    FROM users
),
sessions AS (
    SELECT user_id, event_time,
        SUM(new_session) OVER (
            PARTITION BY user_id ORDER BY event_time
        ) AS session_id
    FROM events
)
SELECT user_id, session_id,
       MIN(event_time) AS start_time,
       MAX(event_time) AS end_time
FROM sessions
GROUP BY user_id, session_id;
```

### Partitioning vs Bucketing — Quick Reference

| Feature          | Partitioning          | Bucketing             |
|------------------|-----------------------|-----------------------|
| Organisation     | Folder structure      | Hashed files          |
| Best for         | Filtering / pruning   | Joins / aggregations  |
| Example column   | `date`, `country`     | `customer_id`, `policy_id` |
| Cardinality      | Low-moderate          | High                  |

### Lambda vs Kappa — Quick Reference

| Feature          | Lambda                        | Kappa                  |
|------------------|-------------------------------|------------------------|
| Pipelines        | 2 (batch + speed)             | 1 (streaming only)     |
| Batch processing | Yes                           | No                     |
| Complexity       | High                          | Low                    |
| Reprocessing     | Re-run batch jobs             | Replay Kafka logs      |
| Tools            | Hadoop + Kafka + Spark        | Kafka + Spark Streaming|
